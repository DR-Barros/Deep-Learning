# Tokenizacin
def tokenize_function(X, y):
    inputs = X.tolist()
    targets = y.tolist()
    model_inputs = tokenizer(inputs, max_length=128, truncation=True, padding=True)
    model_inputs["labels"] = torch.tensor(targets)  # Etiquetas como tensor
    return model_inputs


# Convertir los datos en formato Dataset para HuggingFace

Preguntas
Cual es el principo de LoRA?


    

  • Porque se usan menos parametros?
    • 

  • Que significa el rango del adaptador?
    • 



Cuales son las diferencias sin y con usar LoRA?


    

  • Se puede entrenar sin LoRA?
    • 

  • Cae en la memoria?
    • 

  • Es mas lento/rapido?
    • 

  • Como cambian las performancias?
    • 























Respuestas


    

  1. LoRA es una tcnica para adaptar modelos, para que a la hora de hacer Finetuning se realice con un nmero mucho menor de parmetros en comparacin con el modelo completo. El principio que esta detrs de estop es utilizar aproximaciones de bajo rango, para ajustar solo una parte del modelo.
    2. 



    

  • LoRA usa una aproximacin de bajo rango para modificar las matrices de pesos del modelo y en vez de ajustar todos los parmetros, LoRA introduce adaptadores de bajo rango que solo modifican un subconjunto de los pesos del modelo. Esto swe logra mediante la factorizacin de las actualizaciones de peso en productos de matrices de rango reducido/
    • 

  • El rango en LoRA se refiere al tamao de las matrices de bajo rango que se utilizan para modificar los pesos del modelo original. Si un adaptador tiene un rango pequeo, significa que solo una pequea cantidad de parmetros adicionales se estn entrenando. La idea detrs de esto es que un rango bajo permite capturar las modificaciones necesarias en las representaciones del modelo con un nmero significativamente reducido de parmetros, sin perder la capacidad de ajuste.
    • 



    

  1. La principal diferencia entre entrenar sin y con LoRA es que con LoRA se reduce significativamente la cantidad de parmetros a entrenar y por ende se acelera el entrenamiento y se logra entrenar con menos recursos
    2. 



    

  • La respuesta a esta pregunta depende de los recursos con los que se cuente, pero para una persona comn no se podria debido a las necesidades de hardware para entrenar un modelo con un nmero tan elevado de parmetros lo hace imposible
    • 

  • LoRA generalmente reduce el uso de memoria en comparacin con entrenar todos los parmetros del modelo. Esto se debe a que solo se almacenan y se ajustan las matrices de bajo rango, lo que reduce significativamente la cantidad de parmetros entrenables.
    • 

  • LoRA puede ser ms rpido que entrenar el modelo completo. Esto es porque solo se entrenan los adaptadores de bajo rango, lo que reduce el costo computacional asociado con el entrenamiento de todos los parmetros del modelo.
    • 

  • LoRA ofrece una solucin intermedia entre la eficiencia computacional y el rendimiento. Aunque no siempre alcanza la misma precisin que el entrenamiento completo del modelo, ofrece un buen equilibrio entre eficiencia y rendimiento, especialmente cuando los recursos computacionales son limitados.

Respuestas


    

  • Teoricamente al pasar de una representacin por defecto de 32 bits a una de  8 0 4 bits se puede pasar a un modelo que ocupa 1/4 o 1/8 respectivamente del original sin tener que disminuir la cantidad de parmetros, en la practica con el modelo de 4 bits tuvimos un uso de VRam de 1gb aproximadamente vs que el modelo por defecto (usando LoRA) nos dio un consumo de VRam de aproximadamente 3gb, lo que si bien es significativo no llega a la diferencia esperada
    

    • 

  • El inters de esta tcnica es que permite usar modelos grandes y finetunearlos con recursos limitados sin tener que disminuir el nmero de parmetros a entrenar. Pero el costo de usar esta tecnica es perder precisin de los valores de cada parametros ya que se pasa de ser representados por 32 bits a 4 u 8 bits solamente
